Research pipeline narrative motion

#241 · DESIGN.md rule 4: slow operations get narrative motion, not spinners. Today: 3-phase loader is there, but fields don't fill in as data arrives, sub-stages aren't visible, no per-stage timing, emoji 🔍 loader elsewhere in the surface. This pass fixes all of that.

What changes

1. Field-fill mechanic. The result card appears immediately with skeleton placeholders. As each stage completes, the corresponding fields fill in with a 200ms ease-out reveal. The user sees progress as data, not as a spinner that finally stops.

2. Sub-stage visibility. Stage 2 (Serper) runs 4 parallel searches — reviews / leaders / jobs / product. Show each as a bullet that resolves independently with its count ("6 reviews", "4 leaders") so the user sees the chain actually working.

3. Per-stage timing. When a stage finishes, show its elapsed time ("1.2s") in tabular-numerics next to the stage title. Green on the done check. Free signal that the system is fast.

4. Streaming synthesis. Stage 3 (Claude) streams the intelligence summary into the result card instead of buffering the whole JSON. Cursor blinks while streaming.

5. Honest skip/fail states. Apollo exhausted → stage shows — + "fell back to web" hint. Serper rate limit → specific sub-items show skipped, not failed.

6. Drop the emoji loaders. The 🔍 jopp-loader-icon on job opportunities in the sidepanel goes away — replaced with the same skeleton-shimmer + chain grammar.

Key decisions locked

  • Surface: applies to both sidepanel.js research flow (`renderResearchPhases`) and company.js research refresh flow. Single component, two mount points.
  • Result card appears immediately with skeleton shimmer on all data slots. Scaffolding known from the company record (name, url, logo initial) — everything else is .skel.
  • Three stages: Firmographics (Apollo), Leaders/reviews/jobs/product (Serper), Synthesis (Claude). Names updated from current ("Fetching firmographics" → "Firmographics · Apollo") to show the provider — reads as a chain, not jargon.
  • Per-stage timing in tabular-numerics at row-right. Green on done, tertiary on active (blank until complete), "skipped" text on skipped.
  • Stage 2 sub-items reveal as each Serper search resolves. Shows result count ("6 results") or skipped (rate limit). Matches the existing parallel-search structure in research.js:495.
  • Stage 3 streaming: intelligence summary streams into the stage body (and into the result card's Intelligence slot) as Claude generates. Cursor blinks while streaming. Uses the existing streaming pattern from chat (chatWithFallback) adapted for this endpoint.
  • Field-fill animation: 200ms ease-out opacity + 2px Y-translate as skeleton fades out. No "pop-in" — smooth transition.
  • Skeleton shimmer uses 1.4s linear (stay below 400ms cap because it's continuous ambient motion, not a discrete transition — this is the one exception to the motion budget).
  • Active stage has a tinted bg (--ci-accent-primary-soft) + pulsing ring around the dot (1.4s infinite). Only ONE stage is active at a time.
  • Chain collapse on finish: 2s after final stage completes, the pipeline card collapses to a single header line "Researched in N.Ns · refresh" at the top of the result card, freeing vertical space.
  • Emoji loaders removed: 🔍 jopp-loader-icon in sidepanel.js (job opportunity block) replaced with same skeleton/chain grammar. No spinner anywhere that runs longer than 400ms.
  • Skip/fail states: stages (and sub-items) have a skipped state — dash-dot icon, "skipped" text, inline hint explaining why. No red ✗ for non-errors; only for real hard failures (network completely down).
  • Timing events: timings come from actual network callbacks (not faked by setInterval). Each provider call brackets its own performance.now() and emits a real completion time.

Motion tokens used

  • --motion-xs (120ms) — stage title weight change, color transitions on the dot.
  • --motion-sm (200ms) — stage background tint, field-fill reveal, sub-item enter.
  • --motion-md (300ms) — stage-sub max-height expand, pipeline-card collapse-on-finish.
  • --ease-out — every transition above (arrivals). Linear banned per DESIGN.md.
