#260 Tasks tab redesign

Opp-scoped tasks inside the new shell (#262). Tasks are stored globally and filtered to opportunityId === currentOpp.id; same treatment across Pipeline Overview (#239). Inline-editable rows, grouped by due status, with Coop-suggested tasks folded directly into the list. No typing, no button-heavy chrome — just a list you can live in.

7 — Open questions

Three decisions — resolved

Matt's verdicts from the first mockup review. All three are locked; design above reflects them.

✅ Q1 — Default group state
Matt: yes, collapse Upcoming by default. First-visit shows Overdue + Today expanded, Upcoming collapsed (counted in the header), Done collapsed. Keeps the hero quiet. Disclosure is cheap.
✅ Q2 — Task sourcing & assignment
Matt: drop "assignee" entirely, but differentiate Coop's suggestions by source. Coop is single-user, so no "assign to" concept. But every Coop-suggested task shows why Coop thinks it matters — "Coop · from Amira meeting transcript", "Coop · from Priya email thread (3-day silence)", "Coop · detected inactivity", etc. The source line replaces the vague "Coop suggested" with something diagnostic. See the updated suggestion cards.
✅ Q3 — Dismissal memory scope
Matt: per-opp only, no global learning for now. When Matt dismisses a Coop-suggested task, it disappears from that opp permanently. Coop doesn't generalize the dismissal to other opps — too slippery, feels presumptuous when the situation differs. If global-dismissal utility emerges later, we'll add an explicit "Dismiss this type of task globally" option to the dismiss action. For now: per-opp is the contract.
