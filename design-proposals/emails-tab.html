Open questions
Single-pane vs. two-pane at wider widths?
At >1400px the main column has 1040px to play with after the right rail. Single-pane keeps reading posture good and avoids ping-pong between list and detail — but at ultra-wide it might feel under-dense.
Option A: Always single-pane (current recommendation).
Option B: Adapt — two-pane at >1400px, single-pane below.
Option C: User toggle (list-view / reader-view).
Quoted content — always-collapse or smart?
Today's fallback dumps the entire quote chain raw. Any collapse is a big win. Question is how aggressive.
Option A: Always-collapse any >-prefixed block (current recommendation).
Option B: Smart-collapse — only collapse if >3 quote levels deep; shallow quotes stay inline.
Option C: Never auto-collapse, but mute the text color of quoted lines.
Reply path — Gmail new tab or inline compose?
Today the only option is open in Gmail. Inline compose would be new scope but closes a loop.
Option A: Reply opens Gmail in a new tab — primary (current recommendation).
Option B: Inline compose uses Coop to draft, user sends via Gmail API (needs send scope).
Option C: Hybrid — Coop drafts inline, user copies to Gmail (no send scope needed).
