6Open questions — iteration 2

Open calls before spec (iteration 2)

  1. Goal editing UX. "Edit goals" button opens a modal with per-metric goal inputs — or does each card inline-expand to reveal an editable number? Gut: modal. Modal is cleaner; inline-expand per card is fiddly when you want to update all four goals at once in one pass.
  2. Goal defaults on first run. When a user sets up the dashboard for the first time, what are the default goal values? Generic numbers (15 apps/month, 50 saves/month, 20 outreach/month) — or derived from user's current pace over the prior 30 days? Gut: generic. New users have no pace data; experienced users will override immediately.
  3. Goal met UX. When a goal is hit (e.g. Applications reaches 15/15), does the card celebrate (border turns green, small confetti burst) or just silently render green? Gut: subtle green treatment only. Celebration risks feeling gamified; green border + "Goal met" label on the card is dignified enough.
  4. Tasks hero: default expanded vs collapsible. Hero Today panel is expanded by default. Is it collapsible (via a chevron), or always pinned? Gut: collapsible. Some days you just want to see pipeline immediately. Expanded by default on first load; last-used collapse state persists per session.
  5. Stage editing — in-place vs settings page. The gear icon reveals Rename / Reorder / Delete inline. Reordering might need drag handles. Alt: gear icon navigates to Pipeline Settings page where all stages are listed as a sortable list. Gut: inline with rename + delete; full reorder in Settings.

Reply "ship it" if these calls look right, or name which ones to workshop before spec.
