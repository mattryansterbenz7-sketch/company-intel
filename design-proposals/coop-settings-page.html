Scope note

Career OS 3-column vision — deferred

On hold for this pass

The Career OS restructure (noted in memory) imagines a 3-column Coop-centric layout with a persistent memory rail on the right, pipeline in the middle, and primary nav on the left. This pass keeps the existing 2-column settings shell and only restructures the Coop Settings page contents into tabs.

Rationale: tabs are a low-risk change that can ship inside the current IA without touching any other page. The 3-column vision requires rewriting the sidebar and saved.js chrome — a much bigger swing that deserves its own issue. If this tab structure tests well we can promote Memory / Usage / Personality into full sidebar destinations later.

For Matt

Open questions

1. OpenAI surfacing — first-class or fallback only?
Keep OpenAI as a provider-level option inside the Models tab (fully pickable per task-type), or only ever show it as a fallback chip and make the primary picker Anthropic-only? My default renders it as a peer — feels honest since GPT-4.1 mini is the current chat default.
2. Personality dials — 3 or 4?
Current: Tone / Brevity / Formality. Candidate 4th dial: Confidence (Measured ⟷ Assertive). Feels distinct from tone, but risks overlap with "push back if I ask about a downgrade" in operating principles.
3. Memory tab — raw insights or just snapshot?
Show both (my default) so Matt can delete bad captures, or only show the consolidated profile snapshot and hide the raw list behind an "Advanced" disclosure? Raw list is noisier but gives real control.
4. Usage chart — 30-day default or toggle?
Default: 30-day line chart. Better to let Matt switch between Day / Week / Month at the top right? Day view would surface runaway-cost incidents faster.
