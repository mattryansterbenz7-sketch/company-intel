Editor behavior · iteration 2

WYSIWYG — never see a raw asterisk

Default view is the rendered doc. Users format three ways: selection toolbar (Medium/Notion-style floating bubble on selection), keyboard shortcuts (Cmd+B, Cmd+I, Cmd+K link, Cmd+Shift+1/2/3 for H1/H2/H3), and slash commands (type "/" anywhere). Underlying storage is plain markdown; a "View source" toggle lives top-right for the occasional inspection. No "Write / Preview" tabs — GitHub's pattern is a 2015 UI, and the user should never feel like they're writing code.

Distinction

Activity log vs. Notes

Two surfaces, two jobs. Activity is the timeline of what happened. Notes is the space for how you're thinking about it.

What Matt verdicted

✓
✓ Markdown storage — approved
Matt: good call. Plain markdown in chrome.storage.local. Portable, diffable, AI-native, plays well with future features (export, search, sync).
✓
✓ TOC sidebar — approved (assuming structure)
Matt: works if notes have structure. Structure comes from the WYSIWYG editor — slash commands + shortcuts push users toward proper headings. TOC auto-populates from h1 / h2 / h3 blocks. Flat notes → empty sidebar, graceful degradation.
✓
✓ WYSIWYG — never see raw asterisks
Matt: users shouldn't have to know markdown syntax. Resolution: inline live-render + selection toolbar + keyboard shortcuts + slash commands. Source toggle for the rare case. No "Write / Preview" tabs — GitHub's pattern is a 2015 UI. See the Editor behavior section for the full treatment.
✓
✓ Coop's notes: always visible, or hidden until Coop has something?
Hidden until populated, pinned to the bottom when it exists. Empty "Coop's notes" section on new saves is dead weight. Once populated, it anchors to the bottom of the Notes pane and stays visible while the user scrolls their own notes. Subtle auto-scroll animation on first entry so it's discoverable.
✓
✓ Coop's notes: dismiss in addition to promote
Yes, both actions. Dismiss hides the entry and logs it; Coop uses the signal to learn which kinds of contributions aren't valuable (meeting summaries dismissed frequently → reduce meeting auto-summaries). Promote lifts the content into the user's own notes at cursor position or end-of-doc.
