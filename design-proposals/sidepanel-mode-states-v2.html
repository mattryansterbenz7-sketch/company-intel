Side panel — iteration on your three notes

Structural direction stays (Pipeline Dashboard with Today view). This pass fixes three gaps you called out: (1) the Today / Pipeline toggle disappears when there's nothing detected (no click-fatigue when there's no pipeline context to choose), (2) Coop is docked at the bottom across every state — not just Apply Mode, (3) Apply Mode surfaces snip + screen share as first-class actions, not hidden inside chat.

What changed in v2

Toggle is now conditional. No company detected, no saved entry → no toggle. Today view shows by default; Pipeline option re-appears only when there's pipeline context to choose.
Coop docked bottom across ALL states. Not hidden in "empty" states, not only in Apply Mode. Minimized by default (input bar + tool buttons); expands to half or full on click.
Snip + screen share always accessible. Both tools live in the Coop input toolbar at the bottom of every state — available one click away regardless of detection or mode.
Apply Mode surfaces tool shortcuts as first-class actions. When Apply Mode is active, Coop half-expands with prominent "Snip this question" and "Share screen" actions — plus the toolbar still has both at the bottom.
Coop avatar in top-right of side panel chrome. Tiny persistent reminder that Coop is one click away, consistent with top-bar placement elsewhere in the product.
State count reduced from 8 to 5. Merged states where the only difference was the top chip. The dock + toggle logic adapts; the body doesn't need a dedicated state for every URL permutation.
Killed the cache / stale / refresh concept entirely. v1 had a "Fresh / Stale" state machine with a 24h `researchCache` TTL + "Refresh research" CTA. v2 removes it. The opportunity record IS the source of truth; whatever it shows (stage, context, score, chips) is what renders. No stale badge, no age indicator, no refresh button. Details in the Data freshness note below.

Data freshness — no cache concept

The side panel never shows a "stale" badge, never gates content behind a "Refresh research" button, and never displays an age ("3 days old"). Everything the side panel renders for a saved opportunity — stage, context chip, score, data chips, Next Steps — comes straight from savedCompanies[entryId] at render time. The opportunity record IS the source of truth.

What about the 24-hour research cache? It still exists in researchCache (per CLAUDE.md) — but as an internal optimization for when Coop re-researches a company. It's invisible to the user. The user sees the opportunity's current data, period.

What happens when the user wants to re-research? That's a user-initiated action (click "Research" button in State 2, or "Refresh" in the Coop menu if someone wants it). Not a passive "your data is stale" nudge. No artificial 24-hour trigger.

Toggle logic
When does Today/Pipeline show?
The toggle is a context-aware affordance. Rule: only show it when the user actually has a meaningful choice. On a blank page (no detection, no saved entry), Today is the only useful view — so no toggle.

3 · Remaining open questions

1. Toggle default when there IS detection: Pipeline or Today?
States 2 + 3 default to Pipeline (the detection or entry is the reason they're there). State 4 is user-chose-Today on a detection page. My default: Pipeline, since detection IS the signal that you want pipeline context. But Matt gets the call — if you find yourself toggling to Today more often than Pipeline on detection pages, flip it.
2. Coop dock default size across states — always Minimized, or Apply Mode gets Half?
v2 mockup: default Minimized everywhere EXCEPT Apply Mode, which auto-Half so Coop can greet with a question like "want me to draft this?" Is that too presumptuous? Alternative: Minimized everywhere including Apply Mode, user expands when ready. My default is auto-Half in Apply Mode — the mode is exclusive and the user entered it intentionally, so Coop can be a bit forward.
3. Snip + Share — also promote to Today view first-class?
In v2 they're toolbar icons on Today (available, but not hero-prominent). Apply Mode gets hero action cards. Should Today also get them as first-class action cards somewhere? My default: no — Today should feel calm and context-light. The toolbar is enough. But worth confirming.
4. Top-bar Coop avatar — clickable to toggle full expand, or decorative?
Current sidepanel.js has `#coop-toggle-btn` — the avatar toggles chat open/closed. v2 keeps this. Worth confirming: click avatar → Coop expands to Half (not Full) as primary action.
