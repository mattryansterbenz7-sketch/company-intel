Mode-aware snip behavior

The snip button's crop UI is unchanged. Only what happens after the crop depends on Apply mode state.

StateBehaviorReason
Apply mode ON Crop → auto-send vision message with no text body; Coop responds from image. User has clicked "Help me apply" or landed on an ATS URL. The system prompt already knows to answer in Matt's voice within character limits — typing the question is pure friction.
Apply mode OFF Crop → stages image in _pendingSnip, shows preview pill over input, user adds a note + sends. (current behavior, unchanged) Outside Apply mode, the snip could be for many purposes (explaining a company's website, discussing a screenshot). User needs to type context.
