Sizing rules

Bounds

  • Min height: 260px — enough to show the header, model row, one assistant line, and input. Never shrink below this.
  • Max height: min(75vh, 720px) — respects small windows (laptop screens) and absolute cap on big ones.
  • Width: fixed per preset. No dynamic width.

Grow triggers

  • On user message send — compute content height of .chat-body (existing scrollHeight), add chrome (header + model + input ≈ 150px), clamp to bounds, animate height to that value over --motion-md with --ease-out.
  • On assistant reply complete (streaming finish) — recompute and grow.
  • NOT per-token. Per-token would flicker and feel busy. The body already scrolls internally while streaming; height catches up when streaming ends.

Shrink triggers

  • Only shrink on Clear chat or new-page-visit. Never mid-conversation.

Preset override

  • User picks a preset (minimized / half / full) OR drags the size → dynamic sizing disables for that session (_userSizedThisSession = true). Re-enables on page reload.
  • Override respects user intent. Don't fight them.
