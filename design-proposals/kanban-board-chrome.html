Kanban board chrome audit

[NEW] — Tier 1 shipped the Kanban card. The board around the cards never got a DESIGN.md pass. Column headers are thin, no board-level search/filter, no "add stage" affordance, empty columns are a one-liner. This pass fixes the chrome — cards themselves stay untouched.

What today is missing

  • No board-level toolbar — search, filter, view switcher, group-by, sort all live outside the Kanban view or don't exist. No unified control surface.
  • Thin column headers — 3px colored top-border is the only differentiator; stage name + count blend into the content. Hard to scan at a glance.
  • No inline add-stage — stages are configured in a hidden settings modal. Can't just drop "+ Add stage" at the end of the columns.
  • Empty columns underdeveloped — one line of italic tertiary text "No cards." Nothing else. Each column's purpose should whisper, not just echo.
  • Drag-over states hardcoded — rgba(255,122,89,0.07) instead of var(--ci-accent-primary-soft). Violates the token system.
  • Column collapse is the only per-column action — no quick-edit (rename, recolor, reorder, archive), no column-level filter ("show only this stage").
  • No per-card quick-move — must drag. Keyboard arrows or a right-click "move to…" menu would help.

What this pass ships

  • Board toolbar — search + filter chips + view switcher + add-stage + overflow menu
  • Column headers with stage-dot + uppercase label + pill count + hover-reveal actions
  • Ghost "+ Add stage" column at the end of the columns row
  • Empty-column state using the EmptyBlock grammar (tight variant) from #165
  • Proper drag-over states via tokens (coral-soft bg + 1.5px coral inset shadow)
  • Drop indicator (2px coral pill with end-dots) between cards while dragging
  • Collapsed column treatment — vertical title, still shows count

What stays untouched

  • The Kanban card itself — Tier 1 locked this grammar (see design-proposals/tier-1a-kanban-card.html). Out of scope.
  • The stage schema / status values / color-picker — no data-model changes
  • Stage-reorder logic (adds affordance on column header, but mechanics unchanged)
Recommended: ship the whole chrome pass together — toolbar + column headers + add-stage + empty columns — they're one coherent surface

Key decisions locked

  • Board toolbar is a new surface above the columns — search (left, ~280px, coop-style muted input that unfocuses to --ci-bg-inset), filter chips (All / My court / Overdue / Qualified / Starred — configurable later), view switcher (Kanban / Table), group-by / sort / overflow icon-buttons.
  • Column header grammar: stage-dot (8px circle, --stage-color) + uppercase label (11px/700, 0.04em tracking) + pill count (11px/700 in --ci-bg-raised pill) + hover-revealed action cluster (collapse / more).
  • Stage color moves from a 3px top border to the stage-dot only. Less chrome, more signal. Border is removed.
  • "+ Add card" ghost row at the bottom of each column — transparent by default, reveals on column hover. Click opens an inline composer.
  • "+ Add stage" ghost column at the end of the columns row — 220px wide, dashed border → solid on hover. Click opens the stage config modal (existing).
  • Empty columns use the EmptyBlock (tight) grammar from #165: no glyph, just title ("Nothing here yet.") + hint ("Drop a card when you have a written offer on the table."). Hint copy is stage-specific and lives in the stage definition.
  • Drag-over state: column shell background → --ci-accent-primary-soft, inset shadow of 1.5px --ci-accent-primary. Card-container optional secondary tint. Tokens only; no hardcoded rgba.
  • Drop indicator: 2px coral pill with 6px end-dots, inserted between cards as the drag hovers. Use --motion-xs fade-in.
  • Collapsed column: 44px wide, vertical-text title (rotate 180° writing-mode). Count stays visible horizontally. Hover expands to show the stage name horizontally briefly (optional).
  • Column reorder: grab the column header to drag. Existing behavior stays; visual polish is the whole change.
  • Search filters cards in place — non-matching cards fade to 0.2 opacity (not hidden, so the board structure stays stable). Matching cards highlight with a subtle 1px --ci-accent-amber ring (120ms ease-out).

Surface → file map

  • Board toolbar: new component, mounted above #kanban-board in saved.html. Handlers in saved.js next to renderKanban().
  • Column header restructure: saved.js:2687-2695 — replace the existing header template with the new one. Delete the 3px color top-border; add the stage-dot.
  • Column actions on hover: saved.js:2993+ — extend delegation for `.col-act` clicks (collapse, more).
  • "+ Add card" ghost row: render after .kanban-cards inside each column. Opens an inline composer (reuse existing save flow).
  • "+ Add stage" ghost column: render after the last real column in renderKanban(). Click → existing openStageConfig() or equivalent.
  • Empty column state: replace the .kanban-empty one-liner with .col-empty using the EmptyBlock-tight grammar. Stage definition gets a new emptyHint: string field (optional).
  • Drag-over + drop indicator: replace all hardcoded rgba(255,122,89,...) with var(--ci-accent-primary-soft) + var(--ci-accent-primary). Drop indicator DOM node is injected into .board-cards during dragover, removed on dragleave / drop.
  • CSS: saved.html:368-474 — the whole .kanban-* block. Replace with the new .board-* tree (rename from kanban-* to board-* is a bonus naming cleanup but can stay as kanban-* if the Doer prefers to minimize churn).

Three small calls before spec

  • 1. Naming. Rename classes .kanban-* → .board-* (my pick — board chrome is more accurate, card grammar already calls itself .kanban-card which stays) or keep .kanban-* to minimize churn. Gut: rename.
  • 2. Stage color treatment. Dot-only (my pick — less chrome) vs dot + subtle 1px top accent that matches the dot. Gut: dot-only.
  • 3. Filter chips set. Default chips = All / My court / Overdue / Qualified / Starred (my pick — matches existing action-status + scoring grammar) vs user-configurable from the start. Gut: ship these five, configurable later.
