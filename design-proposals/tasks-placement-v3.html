Key decisions locked

  • Layout: Single Pipeline Overview section, split 50/50. Stats left, tasks right. Divider = 1px var(--ci-border-subtle) vertical line.
  • Section header: kept as "Pipeline Overview" with subtitle showing the date ("Friday, Apr 17 · 72 companies · 34 opportunities"). Period toggle (Daily/Weekly/Monthly) + date picker stay in the section header — they control the stats side. Tasks get their own Today/Week/All scope tabs inside the right column.
  • Stat cards: 2×3 grid (6 slots; "Active In Play" spans 2 columns in the bottom row). Same content as today, tighter layout.
  • Task rows: list-grammar, not chip-grammar. 14px/500 title, 12px tertiary meta. Checkbox = 18px circle. Overdue = red left-rule + red date. Today-due = amber date. Done = green check + strike-through.
  • Hover reveals: three action buttons at row right — snooze (⏱), open-company (↗), more (⋯ for reschedule/delete/edit).
  • Email-suggested tasks: 5px amber dot at the row's left edge (not a border, not a pill). Inline "keep · dismiss" links in meta row until reviewed. Matches existing review flow logic.
  • Done collapse default: hidden. Footer: "N completed today · show" → inline reveal. Click "hide completed" to re-collapse. Preference saved in localStorage.
  • Empty state: "Clear slate." title + "Nothing due today. Add a task →" — keeps "+ Add a task" ghost row visible. No illustration, no motivation copy.
  • + Add a task: ghost row at the bottom of the active list. Click → inline composer (title, company picker, date). Same data model as existing tasks (text, company, dueDate, priority default "normal").
  • Scope tabs: Today (default) / Week / All — tasks-only filter, stored in localStorage. Independent of the Daily/Weekly/Monthly period toggle (which applies only to stats).
  • Narrow viewport: below 1100px, stack to single column — stats on top, tasks below. Keeps parity with mobile behavior.
  • #231: close as subsumed — this ships the whole header restructure.

Three small calls before spec

  • 1. Column split ratio. 50/50 matches the mockup. Alternative: 55/45 stats/tasks (more room for goal cards to breathe), or 45/55 tasks/stats (more room for task title text). Gut-call: 50/50.
  • 2. Section label on the tasks column. "Today" (my pick — implies the scope) vs. "Tasks" (literal, but redundant with the scope tabs). Gut-call: "Today."
  • 3. Keep overdue under "Today" scope. When scope = Today, the mockup includes overdue items at top. Alt: strict "only due today" (overdue moves to a separate row or "Week" scope). Gut-call: include overdue — they're urgent and belong near today.

If my three gut-calls all look right, reply "ship it" and I'll write the PRD and hand off. Otherwise name which ones to change.
