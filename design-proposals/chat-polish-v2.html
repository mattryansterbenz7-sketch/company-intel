B Softer card-per-turn rejected
Each turn would get a subtle #F9FAFB background with gap spacing. Rejected because it violates DESIGN.md rule 4 — one separator, not three. Borders + backgrounds + shadows stack. The panel already has a border and shadow; adding card-per-turn backgrounds makes the chat feel like an admin dashboard, not a conversation.

Pure text, no action chips at all until you hover a message. Rejected because chips are the primary intent-surface. Implementation
Notes & open questions
Behavior
Auto-follow during streaming. Stick scroll position to bottom while the user is at bottom; release it the moment they scroll up (threshold: 40px from bottom). No "scroll to bottom" button unless released, then fade one in.
Streaming
Handoff to #79 — tokens arrive at the pace of thought, not the pace of the API. Throttle render to 15–30 tok/s visual rate even when the wire is faster. Blinking caret at the tail; removes on complete.
Motion
Chip-row reflows (when a new turn arrives and the chip set changes) animate via 200ms ease-out. Individual chips fade + translate 4px. No layout-shift jump when the answer streams in.
Hover timing
Model pill reveals tokens/cost with 150ms delay on hover-in, instant on click. Click toggles persistent expand until user clicks elsewhere. — open: is the persistence too sticky? Test both.
Empty state
Prompts are context-aware — different on a company page vs. meeting page vs. global chat. Copy stored per-surface; falls back to "Research / Cover letter / Interview prep" when no surface context.
Fallback note
Shows under the first reply after a fallback fires, not every subsequent reply in the same session. Clears when user explicitly switches back to primary model.
Out of scope
Streaming visual (rate, caret behavior) is tracked in #79. This issue only specifies where the caret goes and how much real estate streaming takes — not the API handoff itself.
