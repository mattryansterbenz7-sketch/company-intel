Empty states + first-run

#165 · Pre-Josiah launch. Five surfaces the first external user will hit before they have any data. Grammar: typography-first, no illustrations, no motivational copy, honest label + single primary CTA.

The grammar (one pattern, re-used)

Every empty state shares the same structure: small outline glyph (optional) → tight title (what's empty, in plain words) → one sentence of orientation (why it's empty, what to do) → one primary CTA (dark pill button) and optionally one muted secondary link. Three sizes: full (dashboard, integrations), compact (company card, side panel), tight (kanban column). No stock illustrations. No "Welcome 👋" exclamations. No "Your journey starts here." The UI is honest about being empty and points you at the next action.

Why no illustrations

Illustrated empty states are the slop tell. They add ornament without information, cost bytes, and date poorly. DESIGN.md rule: earn every pixel. A well-typed title + a one-line explanation + a clear CTA communicates faster than a decorated ghost holding a magnifying glass.

First-run sequence

First user-launch is a 3-step checklist (Add a key → Tell Coop about yourself → Visit a company) rendered INSIDE the side panel, not a separate onboarding modal. Each step links to the relevant page. Progress persists in chrome.storage.local.

Recommended: ship all five together — they share one grammar, one component, one PRD

Key decisions locked

  • One component, three sizes. Single <EmptyBlock> pattern: glyph (optional) → title → sub → primary CTA → optional secondary link. Sizes: full (dashboard hero, integrations), compact (company card, sidepanel), tight (kanban column hint, table row).
  • No illustrations. Ever. Slop signature.
  • No motivational copy. No "Your journey starts here." No "Welcome! 👋." Plain labels, single sentence of orientation, one action.
  • Icons are outline-only SVGs via single-color currentColor stroke. No filled shapes, no emoji.
  • Primary CTA style: dark-navy pill (--ci-bg-header) with white text. Matches the existing navy chrome. Secondary: outlined, no fill.
  • First-run checklist lives inside the side panel, not a separate modal. Persists in chrome.storage.local under onboardingState. Dismissable.
  • Empty dashboard: goal cards render zeros at tertiary color (not hidden). Kanban columns render with a short hint ("Companies you're tracking") that fades on first card added to that column.
  • Sidepanel no-detect: honest "Nothing to detect here" + current URL in a monospace chip + 3 escape hatches (pipeline / chat / tasks). Day-0 variant swaps those for sample pages.
  • Integrations zero-state coral banner makes required-vs-optional unmissable. Required gets filled pill CTAs, optional gets outlined.
  • Zero-research company card: slots labeled "Not researched" in italic tertiary; one dashed-border CTA at the bottom calls Research. Integrates with #241 when clicked.
  • All motion: 120ms hover, 200ms state change. var(--ease-out). No linear, no decorative pulses.

Surface → file map

  • First-run checklist: sidepanel.js new mode first-run, guarded by onboardingState.completed !== true. Detection suppressed during first-run.
  • Empty pipeline dashboard: saved.js — renders empty block when savedCompanies.length === 0. Kanban columns render hints when empty.
  • Sidepanel no-detect: sidepanel.js — new state mode === 'no-detect' when content.js returns no company. Shows escape-hatch menu.
  • Integrations zero-state: integrations.js / integrations.html — banner above provider list when no keys set.
  • Zero-research company: company.js — slot placeholder + research CTA when entry.employees === undefined and all enrichment fields absent.

Three small calls before spec

  • 1. First-run position. Inline in sidepanel (my pick — less disruptive, context-sensitive) vs. a full-page welcome modal on first launch. Gut: sidepanel inline.
  • 2. Checklist persistence. Dismiss = hidden forever (my pick), or dismiss = snooze-until-next-session? Gut: hidden forever with an option in Settings to re-open.
  • 3. Day-0 sample page suggestions. Include real companies (Stripe on LinkedIn) as samples (my pick — concrete, actionable) vs. generic instruction ("open any LinkedIn company page"). Gut: real samples; links open in new tab.
