Why not Option B or C
Option B — Collapsible sections in main Coop Settings Rejected
One long scrollable Coop Settings page with Apply Mode as a collapsed accordion alongside Models, Personality, and Memory. Rejected because the existing settings page already uses horizontal tabs for clean separation of unrelated config domains. Burying Apply Mode as a collapsed section among models and memory controls dilutes its visibility and breaks the tabbed pattern users already learned. Adding a fifth+ collapsed region turns a scannable settings surface into a wall of accordions.
Option C — Separate Apply Mode page with its own top-level nav Rejected
A dedicated /apply-mode-settings page navigated to from either Coop Settings or directly from the side panel Apply Mode toggle. Rejected because it fragments Coop's configuration surface — Matt would then have two places to look for Coop-related settings (Coop Settings AND Apply Mode Settings). Apply Mode is a Coop feature, not a separate product. The tab pattern keeps the mental model "all Coop configuration lives in Coop Settings" intact.
Open questions
  • 1. Should the Writing assistant appear at all given it's currently shelved? Lean: yes — disabled toggle. It makes the feature discoverable for when it ships, and the "Experimental" tag is honest about its status.
  • 2. Red-pill indicator: should it change visual state (filled vs outline) when Apply Mode is actively running on the current tab, vs always-visible? A filled pulse would mean "on now," while the outline version means "this tab configures it" — worth testing both to see which reads more clearly.
  • 3. Should the deep-link from the side panel open Coop Settings in a new tab or navigate in the side panel itself? The side panel can host full settings UIs (we already do this for Coop Settings from the main nav), but a full-page tab gives more breathing room to configure Quick Links and the Answer Library.
  • 4. Scope of the Answer Library editor — does the "Manage answers" button open an inline editor within the tab, a modal over it, or a full sub-page? Leaning toward inline slide-in within the tab so the context stays on Apply Mode settings.
  • 5. Does the Quick Links row need ordering controls, or inherit order from Personal Info & Links? If we keep it read-only here, re-ordering only happens in Preferences. Simpler surface, but means one more hop for a common tweak.
