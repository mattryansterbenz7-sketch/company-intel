Chat Architecture: Unified Tool-Use

All Coop chat modes route through a single tool-use path. No more legacy split.
ALL MODES G2 Tool-Use Path
Used for: Company chat, global pipeline chat, side panel chat, application helper mode, Career OS editor mode.

How it works: Slim system prompt (~3000 tokens cached base) with standard-tier profile/prefs embedded. Mode-specific instructions (application helper, Career OS editor, entry updates, task creation) injected into the prompt tail. Model gets 5 tools it can call on demand. Loops up to 5 times: model requests tools, handler executes them, feeds results back.

Model: User-selectable via model picker. Default: Haiku 4.5. Supports all Claude + OpenAI models with tool-use (including GPT-4.1 mini/nano).
Prompt caching: System prompt split into base (cached) + tail (mode-specific, not cached). First message writes the cache (~$0.003); follow-ups read at 90% discount.
Screenshots: Attached as base64 image blocks to the last user message when screen sharing is active.
FALLBACK ONLY Legacy Full-Context Path
Used for: Fatal error fallback only. If handleCoopMessageToolUse() throws an unrecoverable error, the system falls through to the legacy path automatically.

How it works: Large system prompt with everything inline (profile, pipeline, company context, emails, meetings). Single API call per message. No tools.

When you'd see this: Essentially never in normal operation. Only activates if the tool-use handler crashes.

How Scoring Works

Hybrid AI + deterministic. AI evaluates evidence against user-configured flags. Deterministic code computes the actual scores. This prevents the AI from inventing numbers.
Inputs
Job description (scraped or fetched from URL/search) + company firmographics + employee reviews + intelligence + emails/meetings/notes + compiled profile .md (full tier) + structured green/red flags with keywords
↓
AI Evaluation (Haiku)
Single prompt asks AI to: (1) evaluate each configured flag against evidence, (2) extract all qualifications from JD, (3) assess comp vs floors, (4) write 1-sentence rationale per dimension, (5) generate role brief + job snapshot. Returns structured JSON.
↓
Deterministic Scoring
Code computes scores. No AI involvement in the math. Post-processes fired flags, validates evidence, strips contradictory or "no evidence" flags, computes per-dimension scores from flag severity deltas, calculates weighted average, applies hard DQ and excitement modifiers.
↓
Output: jobMatch Object
score (1-10), scoreBreakdown (5 dimensions), flagsFired, neutralFlags, qualifications[], compAssessment, dimensionRationale, coopTake, keySignals, roleBrief, jobSnapshot, interactionSummary, hardDQ, scoreRationale.

The 5 Dimensions

Dimension Default Weight How Score is Computed
qualificationFit 20% Deterministic from qualification counts. AI extracts each requirement from JD and marks met/partial/unmet/unknown. Code computes: (met + partial*0.5) / total * 10, clamped [1,10]. Comp-related qualifications are filtered out (handled by compFit).
roleFit 20% Baseline 5.0 +/- fired flag deltas. Green flags in role/other categories add severity * 0.5. Red flags subtract. Clamped [1,10].
cultureFit 25% Same baseline+delta formula. Culture and team category flags. AI must reference Glassdoor/employee ratings when available (4.0+ = positive, 3.0-3.9 = neutral, <3.0 = risk).
companyFit 20% Same baseline+delta formula. Company, industry, and product category flags.
compFit 15% Baseline 5.0 + structural comp adjustments. Base salary vs floor: above_strong +2.5, above_floor +1.5, at_floor 0, below_floor -2.5. OTE similar but smaller deltas. Plus any comp-category flags.
Overall score formula: Weighted average of all 5 dimensions. Then: hard DQ caps at 4 if a severity-5 dealbreaker fires. Excitement modifier nudges toward user's star rating (rating 4-5 nudges up, rating 1-2 nudges down, by 0.3 per point). Final score clamped [1, 10].

Evidence Validation

Deterministic post-processing strips invalid AI flag evaluations.
  • No-evidence filter: If flag evidence matches "no evidence", "no direct evidence", "no mentions", "no indication", "not enough information" — flag is stripped. It did not fire.
  • Contradictory filter: If evidence starts with "not", "no", "doesn't", "does not", "this is not", "they are not" — the AI is arguing against the flag. Stripped.
  • Unknown ID filter: If AI returns a flag ID not in the configured list, silently skipped.
  • Hard DQ validation: AI can flag hardDQ, but code only honors it if a severity-5 dealbreaker actually fired. Otherwise suppressed.
  • Neutral flags: Configured flags marked unknownNeutral: true that don't fire are tracked separately as neutral — zero impact on score.

JD Backfill

If an entry has no jobDescription when scoring starts, the system tries to fetch one before scoring.
  • Step 1: If the entry has a jobUrl, try a direct HTTP fetch. Works for Greenhouse, Lever, Workday, Ashby. Fails for LinkedIn (auth wall).
  • Step 2: If direct fetch fails or no URL, run a Serper search: "Company" "Title" job description responsibilities qualifications.
  • Step 3: If found, backfill entry.jobDescription and persist to storage so future rescores don't re-fetch.
  • Step 4: If nothing found, scoring proceeds with metadata only (firmographics, reviews, intelligence).

Prompt Caching

Anthropic's prompt caching significantly reduces cost for multi-turn conversations.
How It Works
The system prompt is split into two blocks, each with cache_control: { type: 'ephemeral' }:

Block 1 (base): Identity prompt + date + principles + profile summary + tool-use rules. ~3000 tokens. Stable across messages in a session.

Block 2 (tail): Current binding context (which company the chat is bound to). ~50 tokens. Changes only when switching companies.

The Claude API automatically caches any cache_control block that exceeds the model's minimum (Haiku 4.5 = 4096 tokens). COOP_TOOLS definitions (~700 tokens) are sent as the tools parameter, which counts toward the cacheable prefix.
Cost Impact
First message: Cache write costs 1.25x the normal input rate. So ~$0.003 for a typical system prompt.

Follow-up messages: Cache read costs 0.10x the normal input rate. That's an ~88% discount on the system prompt tokens.

In practice: A 10-message conversation where the system prompt is ~3000 tokens saves roughly $0.025 compared to no caching.

Console logging: Each tool-use step logs cacheW (write tokens) and cacheR (read tokens) so you can verify caching is active.

Cost Tracking

All API calls route through claudeApiCall() or openAiChatCall(), which call trackApiCall() for every request.
  • Per-call logging: Every API call logs to console: [Cost] model -- in:X out:Y (cache: write=Z read=W) -> $0.XXXX (today: $X.XX)
  • Daily tracking: apiUsage in chrome.storage.local tracks per-provider daily history (last 30 days), request counts, token counts, estimated cost.
  • Call log: Last 500 calls within 24 hours, with provider, model, token counts, estimated cost, and operation tag (chat/scoring/research/insight/profile/extract).
  • Rate limit headers: For Anthropic, tracks tokens-remaining, tokens-limit, requests-remaining, requests-limit from response headers.
  • View costs: Open pipeline.html#usage to see the usage dashboard with daily breakdown.

Tool Loop

Step 0: Initial API call
Send system prompt + tools + conversation history. Model either responds with text (done) or requests tool calls.
↓ stop_reason: tool_use ↓
Steps 1-4: Tool execution
For each tool_use block in the response: execute runCoopTool(), collect JSON results, append as tool_result messages, send back to model. Model can call multiple tools in parallel within a single step.
↓ stop_reason: end_turn ↓
Final: Text response
Model returns a text block. Extracted as the reply. Usage tracked across all steps (input, output, cacheCreation, cacheRead). Tool call log returned for debugging.
Hard cap: 5 tool-loop iterations. If the model hasn't produced a text response after 5 steps, a fallback message is returned: "I reached the tool-call limit while gathering context."

Result size cap: Tool results are capped at 60,000 chars (~15k tokens) to prevent context overflow.

What's Embedded Inline vs Fetched On Demand

Embedded in System Prompt (every message)
  • Identity block: Coop personality, preset, tone rules
  • Today's date
  • Operating principles from coopConfig
  • Standard profile: coopProfileStandard (~800 tokens)
  • Standard preferences: coopPrefsStandard (~2300 tokens)
  • Tool use rules: 10 rules + patterns + anti-patterns + grounding + discipline
  • Mode instructions (tail): Application helper, Career OS editor, entry-update proposals, task creation — injected per mode
  • Binding context (tail): Which company this chat is about (or "global")
Fetched On Demand (tool calls)
  • Full-tier profile/prefs: Only when drafting, scoring, or deep career analysis
  • Company details: Firmographics, leadership, role data, intelligence, notes
  • Communications: Email threads, meeting transcripts (keyword-expanded)
  • Pipeline overview: All active entries for cross-pipeline questions
  • Memory entries: Past insights and learned facts, searched by keyword

Chat Modes

All modes use the same G2 tool-use handler. Mode-specific instructions are injected into the prompt tail (not cached).

Application Helper Mode

  • Trigger: context._applicationMode: true — detected when the user is filling out a job application form.
  • Behavior: Short, copy-paste-ready answers in the user's voice. On the first application question, Coop auto-fetches get_company_context + get_profile_section(tier: "full") in parallel to draft from full context.
  • Output format: One clean answer, 2-5 sentences, no preamble, no quotation marks.

Career OS Editor Mode

  • Trigger: careerOSChat: true — preferences page chat.
  • Behavior: Can add/update structured profile entries via career-os-update code fences, change system settings via settings-update code fences. Fetches profile via tools when asked about current values.
  • Output format: Explanation + structured code fence for the UI to parse and apply.

Company / Pipeline Chat (default)

  • Trigger: Any chat message that isn't Career OS or application mode.
  • Behavior: Standard tool-use loop. Can propose entry updates (entry-update code fences) and create tasks (create-task code fences).

Legacy Fallback

  • Trigger: Only if handleCoopMessageToolUse() throws a fatal error.
  • Behavior: Falls through to the old monolithic prompt path. Full profile + pipeline + company context + emails + meetings all inline. No tools. Essentially a crash safety net — should never activate in normal operation.

Coop Memory System

Persistent, typed memory store modeled after Claude Code's project memory. Entries are extracted from chat conversations and stored in coopMemory.

Extraction Flow

  • After every chat message: extractInsightsFromChat() queues a Haiku call with 60-second debounce. Multiple rapid messages = 1 extraction call.
  • Blocking extraction: If the user says "remember this", "from now on", etc., extraction runs immediately (blocking) before the chat response returns.
  • Actions: The extraction AI returns { actions: [create/update/delete] }. Applied via applyCoopMemoryActions().
  • Dedup: Name collision on create converts to update. Entries capped at 200.
  • Injection: buildCoopMemoryBlock() formats entries into a MEMORY.md-style block grouped by type. In the G2 path, memory is fetched on demand via the search_memory tool. Legacy fallback injects it inline.

Coop Assist (Writing Helper)

Content script (coop-assist.js) that runs on all pages. Ambient Grammarly-style writing assistant.
  • What it does: Watches focused text fields. Runs local voice heuristics + cached LLM proofread. Surfaces a floating pill with suggestions and rewrite modes.
  • Rewrite modes: "In my voice" (uses voice profile), "Tighten", "Punchier", "Warmer".
  • Model: Haiku 4.5 via coopAutofill task ID.
  • Cost: ~$0.001 per proofread. Only fires on user interaction (clicking the pill), not continuously.
  • Domain blocklist: Disabled on banking, auth, and government sites.
  • Voice profile: Uses voiceProfile.antiPhrases to flag phrases that don't match the user's natural voice.
